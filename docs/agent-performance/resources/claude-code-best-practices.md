# Claude Code Best Practices

*Reference: [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice)*

This guide focuses on scaling agent performance by adopting "agentic engineering" principles and structured orchestration, specifically within the context of coding agents like Claude Code.

## Key Strategies for Maximizing Agent Performance

1. **Command → Agent → Skill Orchestration**
   - Utilize a tiered orchestration pattern for complex workflows. Break down large objectives into a master plan, dispatch specialized sub-agents for isolated tasks, and rely on pre-defined "skills" for routine operations.

2. **Plan Mode and Gated Phases**
   - Always start a task in a "plan mode." Execution should be gated—agents must explicitly propose a plan and have it approved before moving to implementation. This prevents runaway errors and ensures alignment.

3. **Subagent Isolation**
   - Offload sub-tasks to isolated subagents to preserve context. Complex tasks pollute the main context window; subagents start with a fresh context tailored strictly to their specific task, returning only a high-level summary upon completion.

4. **Concise Context Management (CLAUDE.md)**
   - Keep project instructions (like `CLAUDE.md`) brief and focused only on essential setup, commands, and absolute project conventions. Overloading these files degrades the agent's attention on the actual prompt.

5. **Test-Time Compute Maximization**
   - Employ separate agents for implementation and Quality Assurance (QA). A specialized QA agent that reviews and tests code generated by an implementation agent acts as a verification loop, significantly boosting the reliability of the final output (similar to maximizing "test-time compute").

6. **Reusable Skills**
   - Turn repetitive daily actions and complex bash sequences into reusable, parameterized commands or agent "skills" to reduce token usage and the risk of manual typos.
