# alirezarezvani/claude-skills

**URL:** https://github.com/alirezarezvani/claude-skills
**Type:** Mega Skill Library

## What It Is

One of the largest community-driven libraries with 235+ production-ready skills. It is designed to be highly portable, working across Claude Code, Gemini CLI, Cursor, and other agentic environments. It covers everything from technical architecture to business advisory.

---

## What It Does Well

- **Comprehensive Coverage** — Likely the most exhaustive list of `SKILL.md` files available in one place.
- **Portability** — Skills are written in a "clean" format that doesn't rely on specific CLI tools, making them easy to drop into any project.
- **Domain Diversity** — Includes non-engineering skills like "Marketing Strategist," "Legal Reviewer," and "Product Manager," reflecting a broad vision for AI agents.
- **Community Contributions** — A very active repo with frequent PRs from the community adding new specialists.

---

## Weaknesses

- **Lack of Structure** — With 200+ files in a flat structure, discovery and navigation can be difficult without searching.
- **Generic Prompts** — Some skills are very broad and might not provide the "deep" specialist knowledge required for complex debugging.
- **No Verification** — No built-in way to verify if a skill actually works or if its instructions are up-to-date with current tool versions.

---

## What agentfiles Could Learn

| Idea | How to Apply |
|------|-------------|
| **Skill Categories** | Use their taxonomy (DevOps, Architecture, Marketing) to further refine the `agentfiles/` directory structure. |
| **Meta-Skill Discovery** | Create a skill (e.g., `af-skill-finder`) that can search a large library like this and suggest which specialists are needed for a task. |
| **Domain Expansion** | Add a "Business & Research" category to `agentfiles` to handle non-code tasks like market research or documentation strategy. |
